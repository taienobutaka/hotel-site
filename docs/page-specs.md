# Page Specs

各ページの設計要件（目的/構成/主要要素/レスポンシブ）を簡潔にまとめた設計書です。
SP: 〜768px / PC: 1024px以上

## 共通仕様

- ヘッダー/フッターは `src/components/` を使用
- フォント: Shippori Mincho（ローカル配信）
- 主要色: `tailwind.config.mjs` に準拠
- 画像/動画: `public/images` / `public/videos` に配置
- 画面内アンカー: `#access` / `#contact` など

---

## 日本語版

### `/ja/` (TOP)
- 目的: 施設の魅力を要約し、各ページへの導線を提供
- 主な構成:
  - FV（動画/画像）
  - CONCEPT / SERVICES / ROOMS / ACCESS / FAQ
  - RESERVATION / CONTACT
- 主要要素: 予約ボタン、セクションナビ、アクセス情報

### `/ja/rooms`
- 目的: 客室・特徴の紹介
- 主な構成:
  - FV
  - コンセプト/特徴
  - ギャラリー
  - 予約セクション

### `/ja/services`
- 目的: サービス内容の紹介
- 主な構成:
  - FV
  - サービス一覧（サウナ/温泉/食事/オプション）
  - 予約セクション

### `/ja/plans`
- 目的: 宿泊プランの紹介
- 主な構成:
  - FV
  - プラン一覧
  - 予約セクション

### `/ja/contact-confirm`
- 目的: 問い合わせ入力内容の確認
- 主な構成:
  - フォーム内容の確認
  - 送信/戻るボタン

### `/ja/contact-complete`
- 目的: 問い合わせ完了の案内
- 主な構成:
  - 完了メッセージ
  - TOPへの導線

### `/ja/privacy-policy`
- 目的: プライバシーポリシーの提示
- 主な構成:
  - 見出し
  - 本文

### `/ja/terms`
- 目的: 宿泊約款の提示
- 主な構成:
  - 見出し
  - 条項本文

### `/ja/usage-terms`
- 目的: 利用規約の提示
- 主な構成:
  - 見出し
  - 条項本文

### `/ja/faq`
- 目的: よくある質問の提示
- 主な構成:
  - FAQリスト
